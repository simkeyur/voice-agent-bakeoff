VoxArena

VoxArena is a fully-reproducible evaluation arena and benchmarking harness for realtime voice agents. It enables you to run identical scripted conversation turns across different LLM providers (e.g., Google Gemini Live, OpenAI Realtime) and score them side-by-side on latency, tool-call accuracy, and hallucinations.

Database-Backed

Web Control Panel

A self-contained React app that allows you to manage credentials, visually edit prompts, trigger comparison runs, and browse results without writing a single line of code.

Stateless

Headless CLI

A lightweight, file-driven CLI designed for integration directly into CI/CD pipelines. It consumes standard JSON or YAML script definitions and returns JUnit test results.

🚀 Installation & Quick Start

Install the library directly from PyPI. The package automatically ships both the headless evaluation runtime and the bundled React static assets.

pip install voxarena

Boot the web-based control panel. The server will initialize, automatically bootstrap default test scripts and prerecorded audio files into your current directory, and open the control panel UI in your browser:

voxarena ui

🖥️ Web Control Panel UI

The control panel runs locally at http://127.0.0.1:8000. It runs completely self-contained, using SQLite to persist everything. You don't need a local code checkout or a .env file.

Key Features

  • Credential & Model Isolation: Configure API Keys independently. Models can be selected from recommended preloaded realtime models or written in via the custom selector.
  • Interactive Utterances Editor: Manage conversation turns inside a visual grid. Configure prompts and specify check validators (e.g. expected response substring or expected tool name and parameters).
  • Live Visualizer: Track in-flight evaluations. Read real-time transcripts, play back agent audio response files, and verify tool execution logs.

Auto-Bootstrapping: Launching voxarena ui inside an empty directory will automatically unpack the default restaurant reservation script (Saffron Leaf) and its audio assets into the local folder so you can start testing immediately.

⚙️ Headless CLI Engine & Scripting

For regression testing, CI integration, or headless runners, the CLI consumes local scripts and exits with status 0 or 1 based on metrics thresholds.

Running a Script

voxarena run --provider gemini --script ./script/utterances.json --min-tool-accuracy 0.95 --junit report.xml

Conversation Script Format

You can define conversation turns in either JSON or YAML format. In both modes, each turn specifies the phrase to say (or an audio WAV input matching the turn ID) and the expectations to check.

[
  {
    "id": "u01",
    "text": "Hello, is this the Saffron Leaf restaurant?",
    "expect": {
      "response": "Saffron Leaf"
    }
  },
  {
    "id": "u02",
    "text": "Can I book a table for tomorrow at seven pm?",
    "expect": {
      "tool": "book_table",
      "args": {
        "time": "19:00",
        "guests": 2
      }
    }
  }
]
- id: u01
  text: "Hello, is this the Saffron Leaf restaurant?"
  expect:
    response: "Saffron Leaf"
- id: u02
  text: "Can I book a table for tomorrow at seven pm?"
  expect:
    tool: book_table
    args:
      time: "19:00"
      guests: 2

Expectation Parameters

The expect block supports several keys for automated grading:

  • response: The substring expected to be present in the agent's textual response transcript.
  • tool: The name of the function/tool the agent should invoke during this turn.
  • args: A dictionary of key-value arguments that must match the tool call parameters. Arguments are evaluated using fuzzy matching to avoid strict type/spacing mismatch issues.

📊 Core Benchmarking Metrics

VoxArena collects and calculates the following key metrics for each agent session:

  • Time-To-First-Audio (TTFA): The duration (in ms) from the moment the user's audio injection completes to the millisecond the first audio frame chunk is returned by the provider.
  • Interruption Stop Latency: The speed (in ms) at which the provider halts speaking when the user injects a turn that interrupts active agent playback.
  • Tool Call Accuracy: The ratio of correct tool invocations (name matches and argument payloads match expected fields) over total expected tool calls.
  • Hallucinations: Verified using automated LLM grading (backed by GEMINI_EVAL_MODEL or OPENAI_EVAL_MODEL) that compares agent transcript statements against a static verified knowledge base.

🛠️ Interactive CLI Command Builder

Adjust the options below to configure your headless benchmark command, then copy the generated command line to run in your terminal or integrate into your GitHub Actions workflow.

voxarena run --provider gemini --transport direct-injection --script ./script/utterances.json --min-tool-accuracy 0.9 --max-avg-ttfa-ms 1800 --max-hallucinations 0 --junit voxarena-junit.xml

🧩 Extensibility (Custom Pipecat Adapter)

VoxArena is built on top of the Pipecat framework. You can add new adapters or test custom LLM pipelines by implementing the adapter class interface.

Create a class inheriting from BaseProviderAdapter and define the LLM service hook:

from voxarena.providers.base import BaseProviderAdapter

class CustomProviderAdapter(BaseProviderAdapter):
    def get_llm_service(self):
        # Instantiate and return your custom Pipecat LLM Service
        return MyCustomLLMService(
            api_key=self.api_key,
            model=self.model,
            # ...
        )
